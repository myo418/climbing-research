# リサーチWebサイト 要件定義（2サイト構成）

作成: 2026-04-21
対象プロジェクト: 後藤ゼミ リサーチ「クライミングをする人々／クライミングと怪我」

リサーチに関連するWebサイトは **2つ** ある。性格も読者も別なので、別プロジェクトとして
構築する。ただしデザインシステム・コンポーネント・素材は共有する。

| サイト | 通称 | ディレクトリ | 読者 | 性格 |
|---|---|---|---|---|
| **リサーチサイト** | `site-research/` | 内部 | 後藤教授 / ゼミメンバー | 作業場 / 全素材・未精査を含む |
| **公開サイト** | `site-public/` | 公開 | インタビュー対象者 / 一般 | 信頼形成 / 精査済みのみ |

---

## 共通事項（両サイト）

### 技術スタック
- Astro + MDX
- React アイランド（必要時のみ）
- CSS Modules + CSS変数（[DESIGN.md](DESIGN.md)のトークン）
- Vercelホスティング（プロジェクトは各サイト1つずつ）
- git pushで自動デプロイ

### 共有リソース
- `DESIGN.md` のトークン・コンポーネント仕様
- `shared/` 配下の実装済みコンポーネント
- `climbing-and-injury/` などのテーマ別md（素材）
- `videos.md` の動画インベントリ

### AIアシスト要件
- ソースは全てテキスト（md / mdx / ts / astro）
- Claudeが直接Read/Edit可能
- 新しい情報を入れるときは素材mdに追記 → 必要なら各サイトに反映

## Part 1. リサーチサイト（`site-research/`）

### 1-1. 目的
進行中のリサーチを**後藤教授とゼミメンバーに常時見える状態**で置く作業場。
- 精査前の文章、途中の仮説、読書メモ、生のインタビュー抜粋を含めてよい
- 思考の過程が辿れる
- ゼミ討議の共通参照点になる

### 1-2. 読者
- 後藤教授
- ゼミメンバー
- 自分（思考の整理先として）

### 1-3. アクセス
- **限定公開（URLを知る人のみ）**: パスワードなし、SEO非表示 (`noindex`)
- 最初はVercelの標準URL（`*.vercel.app`）で配布
- 将来的にBasic認証等を検討する余地は残す

### 1-4. コンテンツ方針
- `research/` 配下の**全md/mdxを自動でページ化**する
- 下書き・未完成・仮説・ボツ案も含めて良い
- 各ページに「ステータス」メタ情報（`draft` / `wip` / `reviewed`）を持たせる
- 秘匿すべきもの（実名インタビュー未承諾分など）は個別に`private: true` フラグで除外

### 1-5. 機能要件
- [ ] 全テーマ横断のサイドバーナビ（フォルダ階層そのまま）
- [ ] 全文検索
- [ ] バックリンク表示（どのページからリンクされているか）
- [ ] タグ・ステータスでの絞り込み
- [ ] 更新日時・最終編集者（git情報）
- [ ] ページ間の関連リンク（手動 or 自動）
- [ ] 図解（Mermaid、SVG、Figma書き出し）の埋め込み
- [ ] 目次（h2/h3から自動生成）

### 1-6. デザイン方針
- **機能優先**。Obsidian Publish的な読み物ハブの見た目
- DESIGN.mdのトークンは使うが、展示的な余白は必要なし
- 一覧性と検索性が最優先
- サイドバー常設、モバイルは折りたたみ

### 1-7. ページ構造（案）
```
/                       ← 現時点の問い、主要テーマへの導線、最近の更新
/climbing-and-injury    ← テーマページ
/climbing-spot          ← 同
/[theme]                ← 同
/tags/[tag]             ← タグ別一覧
/status/draft           ← ステータス別（下書きだけ見る等）
/search                 ← 全文検索
```

## Part 2. 公開サイト（`site-public/`）

### 2-1. 目的
- リサーチの**精査された知見・作品を提示**する
- **インタビュー対象候補に「この研究は何者か」を伝え、信頼してもらう**
- 最終的な美大ゼミの成果物として機能させる

### 2-2. 読者
- これからインタビューを依頼するクライマー
- インタビュー済みの協力者（自分の扱われ方を確認できる）
- 後藤ゼミ外の教員・学生
- 将来の閲覧者（ポートフォリオ的に）

### 2-3. アクセス
- **一般公開**
- カスタムドメイン（後で取得）
- SEO許可（`index`）
- SNSシェア時のOGP最適化

### 2-4. コンテンツ方針
- **精査済みの情報のみ**を載せる
- 匿名化・当人確認を済ませたインタビューのみ掲載
- 仮説を書く場合も**明確に「仮説である」と明示**する
- 出典を必ず添える（引用・画像・統計すべて）
- 誤りが見つかったら訂正履歴を残す

### 2-5. 必須ページ（信頼形成のため）
- [ ] **About / リサーチの目的** — 何を知りたくて、どう進めているか
- [ ] **研究者について** — 自分の立ち位置、クライミング歴、利害関係
- [ ] **研究倫理と公開方針** — 匿名性、掲載前確認、撤回ポリシー、データ保管
- [ ] **インタビュー協力のお願い** — 何を聞きたいか、所要時間、掲載前の確認フロー
- [ ] **連絡先** — メール or フォーム

### 2-6. 機能要件
- [ ] レスポンシブ（モバイル・PC）
- [ ] OGP自動生成（タイトル・サムネ）
- [ ] 記事間の「次を読む」導線
- [ ] 参考文献のまとめページ
- [ ] RSS（任意）
- [ ] 問い合わせ導線（メール or フォーム）

### 2-7. デザイン方針
- **展示物レベルの強度**。DESIGN.mdを厳密に適用
- 余白多め、タイポグラフィ中心、写真は選び抜く
- 色は静かに、アクセントは意味ある時だけ
- 1画面に1主題。スクロールで読ませる

### 2-8. ページ構造（最小）
```
/                       ← リサーチの導入、主要章への導線
/about                  ← About / 目的
/about/researcher       ← 研究者について
/about/ethics           ← 研究倫理・公開方針
/about/interview        ← インタビュー協力のお願い
/contact                ← 連絡先
/essays                 ← 読ませる文章群（ここに章ごとのページを増やしていく）
/essays/[slug]          ← 個別エッセイ
/voices                 ← インタビュー（承諾済・精査済のみ）
/voices/[slug]          ← 個別インタビュー
/sources                ← 参考文献・出典
```

### 2-9. 倫理・法務メモ
- 肖像権: インタビュー対象者の顔写真は文書で許諾を取る
- 引用: 文献・映画・写真のフェアユース範囲を守る
- 匿名: 実名か仮名かを本人が選べる
- 撤回: 本人が公開取り下げを申し出たら即座に対応
- 未成年: 現時点では対象外の想定（変わったら別途ポリシー整備）

### 素材mdとサイトの関係

- **`research/[theme]/*.md`** が素材の源泉
- **`site-research/`** は**素材mdを全部取り込む**（自動的にページ化）
- **`site-public/`** は**`site-public/src/content/` に自前で書いたMDX**のみ表示
  - 素材を参照しつつ、公開用にリライト・構成しなおす
  - 直接素材をレンダリングしない（精査が挟まるため）

## 運用ルール

### 素材mdへの書き込み（素材源泉）
- 自由に書いて良い（思考、引用、仮説、疑問）
- `site-research/`に自動的に出るので、ゼミメンバーにも見える
- 絶対に外に出したくない個人情報は `climbing-and-injury/private/` 等に隔離

### 公開サイトへの反映
- 素材mdから内容を選び、**精査し、書き直して** `site-public/src/content/` に配置
- インタビュー系は対象者の掲載承諾を取った後に反映
- 出典・引用元を明記

### デプロイ
- git push で両サイトが自動デプロイされる
- 両方とも別Vercelプロジェクトで管理
